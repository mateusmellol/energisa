React é uma biblioteca JavaScript pra construir interfaces. Criada pelo Facebook em 2013, hoje é a tecnologia de frontend mais usada no mercado.

A ideia central é simples: ao invés de construir uma página inteira do zero toda vez que algo muda, você divide a interface em **componentes** — pedaços reutilizáveis e independentes. Um botão é um componente. Um card de produto é um componente. O menu de navegação é um componente.

Pensa assim: no Figma, você cria um componente de botão e usa ele em 30 telas. Se mudar o estilo do botão no componente mãe, muda em todo lugar. React funciona com a mesma lógica — só que no código que roda no navegador.

**Por que isso importa pra você como designer?**

Quando você trabalha com um time que usa React, os componentes do Figma têm uma correspondência direta com os componentes do código. Um design system bem feito no Figma vira um design system no React. Isso significa que quando você fala "componente de card" ou "variante do botão", o dev entende exatamente o que você quer dizer — a linguagem se alinha.

Também muda o que é viável pedir. Num site estático, cada estado diferente da interface precisa de trabalho manual. No React, estados (hover, loading, erro, sucesso) são nativos. Você pode pensar e projetar em estados sem medo de estar pedindo o impossível.

**O que React não é:**

Não é um framework completo pra construir sites. É só a parte da interface. Por isso ele costuma aparecer junto com outras coisas: Next.js (que adiciona roteamento e performance), Tailwind (que cuida do estilo), Supabase (que cuida dos dados). Quando você ouve "stack React", provavelmente tem mais coisas embaixo.

**Resumo prático:**

React = interface feita de peças reutilizáveis, que reage a mudanças sem recarregar a página. É o padrão do mercado. Saber o nome e a lógica básica já coloca você num nível de conversa diferente com qualquer dev frontend.
